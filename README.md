# أرشيفي 🎬

موقعك الشخصي لتتبع المسلسلات والأفلام والأنمي والمسرحيات، مع بحث يومي تلقائي عن الأخبار (تجديد مواسم، إلغاء، تأجيل، مواعيد عرض) عبر GitHub Actions — مجاني بالكامل.

---

## 1) كيف تنشر الموقع (مرة واحدة فقط)

لا تحتاج أي تسجيل أو حساب أو مفتاح API — كل مصادر البيانات (TVmaze وiTunes) مجانية بالكامل وبدون أي اشتراك.

1. أنشئ مستودع (Repository) جديد على GitHub، **عام (Public)** — مهم لأن GitHub Pages المجاني يحتاج مستودع عام، واسمه مثلاً `watchlist`.
2. ارفع كل ملفات هذا المشروع بهيكلتها كما هي (نفس المجلدات: `.github`, `data`, `scripts`, و `index.html`).
   - أسهل طريقة: من صفحة المستودع الفاضي، اضغط "uploading an existing file" واسحب كل الملفات والمجلدات.
3. روح لـ **Settings → Pages**، وفي "Build and deployment" اختر **Deploy from a branch**، الفرع `main`، المجلد `/ (root)`، واحفظ.
   - بعد دقيقة أو دقيقتين بيظهر لك رابط موقعك بصيغة `https://username.github.io/watchlist/`.

## 2) أول تشغيل

1. روح لتبويب **Actions** في مستودعك، فعّل الـ Workflows إذا طلب منك ذلك.
2. شغّل **"تحديث يومي (Daily Update)"** يدويًا أول مرة (Run workflow) للتأكد إن كل شي شغّال.
3. من الآن، بيشتغل تلقائيًا كل يوم بدون أي تدخل منك.

---

## كيف تضيف عمل جديد

1. روح لتبويب **Actions** في مستودعك.
2. اختر workflow اسمه **"إضافة عمل (Add Item)"** من القائمة الجانبية.
3. اضغط **Run workflow**، عبّي:
   - **title**: اسم العمل (عربي أو إنجليزي)
   - **type**: `series` / `movie` / `anime` / `play`
   - **year_hint** (اختياري): سنة الإصدار، تساعد لو فيه أكثر من نتيجة بنفس الاسم
4. اضغط **Run workflow**. بعد دقيقة تقريبًا بيظهر العمل في موقعك (حدّث الصفحة).

أو من داخل الموقع نفسه: اضغط **"إضافة عمل"** في الأعلى — بيفتح لك هذي الصفحة مباشرة بعد ما تربط اسم مستودعك مرة واحدة (⚙ في الشريط الجانبي).

### المسرحيات
لا توجد قاعدة بيانات مجانية بدون تسجيل تغطي المسرحيات، فستُضاف بدون بوستر أو معلومات تلقائية. تقدر تضيف رابط بوستر ووصف يدويًا من تبويب "نظرة عامة" داخل تفاصيل العمل في الموقع (يُحفظ في متصفحك).

## كيف تحذف عمل

1. من تفاصيل العمل في الموقع، اضغط **"حذف نهائي من المصدر"** — يفتح لك ملف `data/titles.json` على GitHub جاهزًا للتعديل.
2. احذف السطر (الكائن) الخاص بهذا العمل، واحفظ (Commit).
3. بيختفي من الموقع تلقائيًا بعد التحديث اليومي القادم، أو شغّل "تحديث يومي" يدويًا لتطبيقه فورًا.

أو اضغط **"إخفاء من أرشيفي"** لإخفائه في متصفحك فقط بدون حذفه من المصدر.

---

## كيف تعمل الأخبار اليومية

كل يوم (الساعة 6 صباحًا تقريبًا بتوقيت السعودية، تقدر تغيّرها من `.github/workflows/daily-update.yml`):

- **TVmaze** (للمسلسلات والأنمي): يتحقق من حالة العمل (مستمر/منتهي) وموعد الحلقة القادمة. إذا تغيّر أي منهما عن آخر فحص، يُنشئ خبرًا تلقائيًا — هذي أدق طريقة لرصد التأجيل أو التجديد. (ملاحظة: TVmaze ما يميّز "الإلغاء" عن "الانتهاء الطبيعي" بشكل مستقل، فكلاهما يظهر كـ "منتهي").
- **iTunes Search** (للأفلام): يجلب البوستر والوصف وسنة الإصدار فقط، بدون حالة أو مواعيد (الأفلام عمل واحد، ما تحتاج تتبّع مواسم).
- **Google News**: يبحث عن اسم كل عمل ويصنّف النتائج بكلمات مفتاحية إلى: تجديد/إلغاء، مواعيد/تأجيل، أو عام. (تصنيف مبسّط مجاني، وليس بدقة تحليل ذكي — قد يفوّت أو يصنّف خبرًا بشكل غير دقيق أحيانًا).

## بياناتك الشخصية

- تقدّمك (الحلقات، الموسم، التقييم، الملاحظات، حالة المتابعة) محفوظ في متصفحك فقط (localStorage) — لا يُرفع لـ GitHub ولا يراه أحد.
- إذا تستخدم أكثر من جهاز/متصفح، استخدم **"تصدير بياناتي"** من الشريط الجانبي لحفظ نسخة، و **"استيراد بياناتي"** على الجهاز الآخر.
- **"تصفير بياناتي المحلية"** يمسح هذي البيانات فقط من متصفحك، ولا يمسح أرشيفك على GitHub.

## ملاحظات مهمة

- المستودع لازم يكون **عام (Public)** ليعمل GitHub Pages مجانًا، يعني أسماء الأعمال اللي تتابعها ستكون مرئية لمن يزور المستودع (بياناتك الشخصية مثل تقدّمك وتقييمك تبقى خاصة في متصفحك).
- لا تجرّب فتح `index.html` بالنقر المزدوج من جهازك مباشرة — المتصفح يمنع تحميل `data/catalog.json` بهذي الطريقة. جرّبه محليًا بأمر مثل `npx serve .` داخل مجلد المشروع، أو فقط انشره على GitHub Pages وافتح الرابط.
- تكلفة هذا المشروع: صفر — GitHub Pages وActions وTVmaze وiTunes وGoogle News كلها مجانية للاستخدام الشخصي بهذا الحجم، وبدون أي تسجيل حساب في أي منها.
