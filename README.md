# أرشيفي 🎬

موقعك الشخصي لتتبع المسلسلات والأفلام والأنمي والمسرحيات والقصص والألعاب، مع بحث يومي تلقائي عن الأخبار (تجديد مواسم، إلغاء، تأجيل، مواعيد عرض) عبر GitHub Actions — مجاني بالكامل، وبدون أي تسجيل أو حساب أو مفتاح API.

## الأقسام والمصادر

| القسم | مصدر البيانات التلقائي | الحالة (مستمر/مكتمل/غير منتهي/تكنسل) |
|---|---|---|
| مسلسلات | TVmaze | تلقائي + كشف تغييرات |
| أنمي | TVmaze (أو iTunes للأفلام السينمائية) | تلقائي + كشف تغييرات |
| أفلام | iTunes Search | غير مطبّق (لا تحتاج) |
| مسرحيات | لا يوجد — يدوي بالكامل | يدوي |
| قصص | لا يوجد — يدوي بالكامل | يدوي |
| ألعاب | لا يوجد — يدوي بالكامل | يدوي |

الأوصاف وعناوين الأخبار تُترجم تلقائيًا للعربية عبر خدمة ترجمة مجانية (أفضل مجهود، قد تكون الترجمة غير دقيقة 100% في بعض الأحيان).

## 1) كيف تنشر الموقع (مرة واحدة فقط)

1. أنشئ مستودع (Repository) جديد على GitHub، **عام (Public)** — مهم لأن GitHub Pages المجاني يحتاج مستودع عام.
2. ارفع كل ملفات هذا المشروع بهيكلتها كما هي (نفس المجلدات: `.github`, `data`, `scripts`, و `index.html`).
3. روح لـ **Settings → Pages**، اختر **Deploy from a branch**، الفرع `main`، المجلد `/ (root)`، واحفظ.
4. بعد دقيقة أو دقيقتين بيظهر لك رابط موقعك بصيغة `https://username.github.io/repo-name/`.

## 2) أول تشغيل

روح لتبويب **Actions** → شغّل **"تحديث يومي (Daily Update)"** يدويًا (Run workflow) أول مرة. هذا الملف يحتوي على 441 عمل جاهز (من قوائمك)، فالتشغيل الأول قد يأخذ من 10 إلى 30 دقيقة — هذا طبيعي. من بعدها، يشتغل تلقائيًا كل يوم بدون أي تدخل.

## كيف تضيف عمل جديد

### الطريقة 1: مباشرة من الموقع (الأسرع)
1. اضغط ⚙ بالشريط الجانبي (أو من زر "إضافة عمل" أول مرة)
2. اربط اسم مستودعك (`username/repo`)
3. أنشئ [Fine-grained Personal Access Token](https://github.com/settings/personal-access-tokens/new):
   - **Repository access** ← Only select repositories ← اختر مستودعك فقط
   - **Permissions** ← Contents: Read and write
   - **Permissions** ← Actions: Read and write
   - حدد تاريخ انتهاء (مثلًا سنة)، وأنشئه، والصق القيمة في خانة Token
4. من الآن، زر "إضافة عمل" يفتح نموذج بسيط داخل الموقع نفسه — تكتب الاسم وتختار النوع وتضغط إضافة، ويُحفظ في GitHub تلقائيًا بدون مغادرة الموقع

⚠️ **تنبيه أمان:** التوكن يُحفظ في متصفحك فقط (localStorage) ولا يُرسل لأي مكان غير GitHub مباشرة. لا تستخدمه على جهاز عام/مشترك، ولا تشارك صورة شاشة لإعدادات المتصفح تظهر فيها قيمته. زر "تصدير بياناتي" لا يضمّن التوكن في النسخة الاحتياطية لهذا السبب.

### الطريقة 2: من تبويب GitHub Actions (بدون توكن)
نفس خطوات سابقًا — تفتح workflow "إضافة عمل (Add Item)" وتعبّي النموذج هناك يدويًا.

### المسرحيات/القصص/الألعاب
لا توجد قاعدة بيانات مجانية بدون تسجيل تغطي هذي الأنواع، فتُضاف بدون بوستر أو معلومات تلقائية. أضف رابط بوستر ووصف يدويًا من تبويب "نظرة عامة" داخل تفاصيل العمل (يُحفظ في متصفحك).

## كيف تحذف عمل
من تفاصيل العمل، اضغط **"حذف نهائي من المصدر"** — يفتح لك ملف `data/titles.json` على GitHub، احذف السطر الخاص بالعمل واحفظ. يختفي تلقائيًا بالتحديث القادم. أو اضغط **"إخفاء من أرشيفي"** لإخفائه في متصفحك فقط.

## تتبّع المشاهدة بسرعة
على كل بطاقة فيه زرّان سريعان بدون فتح العمل: **✓** (شاهدته) و **↩** (سحبت عليه) — ضغطة واحدة تكفي. للتحكم الدقيق (الموسم/الحلقة/التقييم/الملاحظات) افتح تفاصيل العمل → تبويب "تقدّمي".

كذلك شارة الحالة (مستمر/مكتمل/غير منتهي/تكنسل) تظهر مباشرة على البوستر بدون فتح العمل.

## استيراد دفعة بيانات (الملف المرفق `local-import.json`)
هذا الملف يحتوي على تقدّمك المُستخلص من قوائمك الأصلية (حالة كل عمل: مكتمل/أتابعه الآن/سحبت عليه، وعدد الأجزاء، وملاحظات). بعد رفع `titles.json` وتشغيل أول تحديث:
1. افتح موقعك
2. اضغط **"استيراد بياناتي"** بالشريط الجانبي
3. اختر ملف `local-import.json` المرفق

ملاحظة: التصنيف آلي بأفضل مجهود من النص الأصلي، راجع العناصر المهمة عندك للتأكد، خصوصًا الأسطر اللي فيها أكثر من إشارة بنفس السطر.

## كيف تعمل الأخبار والحالة اليومية
- **TVmaze** (مسلسلات/أنمي): يتحقق من الحالة وموعد الحلقة القادمة، ويولّد خبرًا تلقائيًا عند أي تغيّر.
- **iTunes Search** (أفلام): بوستر ووصف فقط.
- **Google News**: يبحث ويصنّف الأخبار، ويُترجم العناوين الإنجليزية للعربية تلقائيًا. أي خبر فيه كلمة إلغاء يفرض تحديث الحالة إلى "تكنسل" تلقائيًا.
- التصنيف مبسّط بكلمات مفتاحية مجانية، وليس بدقة تحليل ذكي — قد يخطئ أحيانًا.

## بياناتك الشخصية
محفوظة في متصفحك فقط (localStorage) — لا تُرفع لـ GitHub. استخدم **تصدير/استيراد** للنقل بين الأجهزة. **تصفير بياناتي المحلية** يمسح هذي البيانات فقط، ولا يمسح أرشيفك على GitHub.

## ملاحظات
- المستودع لازم يكون **عام (Public)** ليعمل GitHub Pages مجانًا.
- لا تفتح `index.html` بالنقر المزدوج مباشرة — جرّبه بأمر مثل `npx serve .` أو انشره على GitHub Pages.
- التكلفة: صفر بالكامل.
